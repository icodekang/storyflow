# StoryFlow AI 视频生成项目 — 开发任务总览

> 项目目标：输入 story 文本，自动调度多个 AI Agent 协作，最终输出 AI 生成的视频。
> 核心参考：电影制作流程（前期策划 → 分镜制作 → 角色设计 → 视觉生成 → 后期剪辑 → 合成输出）
> **核心框架：pi-mono** — 项目所有 AI Agent 的运行时核心

---

## 一、核心技术理念

### 为什么是 pi-mono

pi-mono 是本项目的**决策引擎**——每个 Agent 的每次执行都发生在 pi-mono 的上下文中。

参考 OpenClaw 的设计理念（其底层使用 `@mariozechner/pi-agent-core`）：

| 层次 | 作用 | 类比 |
|------|------|------|
| **pi-mono Core** | Agent 运行时、Session 管理、记忆上下文 | 操作系统 |
| **Memory Layer** | 短/中/长期记忆检索与存储 | 大脑皮层 |
| **Tool Layer** | Agent 可调用的工具集 | 手和脚 |
| **Orchestrator** | 多 Agent 协作与流程编排 | 大脑前额叶 |
| **Pipeline Agent** | 具体业务 Agent（分镜、角色设计等） | 专家 |

pi-mono 让每个 Agent 拥有：
- **持久 Session**：跨任务保持上下文
- **分层记忆**：短期（消息历史）→ 中期（任务记忆文件）→ 长期（向量检索）
- **工具抽象**：统一工具注册与调用协议
- **事件总线**：订阅/发布 Agent 执行状态

## 三、pi-mono Core 详解

### 3.1 Session 管理

每个 story 生成会话 = 一个 pi-mono Session。

```python
class Session:
    session_id: str          # 唯一标识 (UUID)
    story_id: str            # 当前 story ID
    created_at: datetime
    status: SessionStatus     # running | paused | done | error
    current_agent: str | None # 当前运行的 Agent
    memory: MemoryLayer       # 关联的记忆层
```

Session 由 `SessionManager` 统一管理，支持：
- 创建 / 恢复 / 中止 Session
- Session 状态持久化（Redis 或文件）
- 并发 Session 调度（默认最多 3 个并行）

### 3.2 Memory Layer（三层记忆）

| 层级 | 存储介质 | 生命周期 | 用途 |
|------|---------|---------|------|
| **短期记忆** | Agent 消息列表（in-memory） | 单次 prompt 周期 | 当前任务的上下文窗口 |
| **中期记忆** | Session 文件（JSON/Markdown） | Session 生命周期 | 本次 story 生成完整记录 |
| **长期记忆** | 向量数据库（ChromaDB/Qdrant） | 持久化 | 跨 story 检索相似经验 |

**中期记忆文件结构**（每个 Session 一个文件）：
```
data/memory/sessions/{session_id}.json
```

**长期记忆检索流程**：
```
Agent.execute()
  → memory.search(session_id, query, top_k=3)
  → 向量相似度检索
  → 注入相关记忆到 prompt
  → Agent 执行
  → memory.add(session_id, story_id, agent_name, output)
```

### 3.3 Tools Registry

统一工具注册表，所有 Agent 共享工具集：

| 工具名称 | 功能 | Agent 用途 |
|---------|------|----------|
| `memory_search` | 长期记忆语义检索 | 跨 story 查相似经验 |
| `memory_get` | 读取中期记忆文件 | 获取本次 Session 上下文 |
| `video_generate` | 调用视频生成服务 | VisualGen / VideoAssembly |
| `storage_save` | 保存文件到 Storage | 存储生成的图片/视频 |
| `llm_call` | 调用 LLM 生成内容 | 各 Agent 核心能力 |
| `human_intervene` | 暂停等待人工 | 人工干预模式入口 |

### 3.4 Orchestrator（编排器）

参考 OpenClaw 的事件驱动 Agent 模式：

```python
class Orchestrator:
    """多 Agent 协作编排器"""

    def __init__(self, session: Session, mode: Literal["auto", "human"]):
        self.session = session
        self.mode = mode

    async def run_pipeline(self, story_id: str, story_data: dict):
        """执行完整 Agent Pipeline"""
        for agent_class in PIPELINE_AGENTS:
            agent = agent_class(session=self.session)

            # 1. 检索记忆
            context = await self.session.memory.search(query=story_data["text"])

            # 2. 执行 Agent
            output = await agent.execute(input_data=story_data, context=context)

            # 3. 写入记忆
            await self.session.memory.add(agent.name, output)

            # 4. WebSocket 推送状态
            await self.ws.send({"type": "agent_done", "agent": agent.name, "output": output})

            # 5. 人工干预模式：暂停等待
            if self.mode == "human":
                await self.wait_for_intervention(story_id, agent.name)
```

## 四、前端开发任务

### 页面结构（无登录）

```
/               → 主工作台
  ├── Story 输入区（支持多 story 卡片）
  ├── 模式切换（全自动 / 人工干预）
  ├── Agent 流程可视化（7 步横向流程图）
  ├── 干预抽屉（右侧滑出，人工干预模式）
  └── 视频预览 / 下载区
```

### UI 设计规范

- **风格**：深色主题（#0a0a0f 背景），极简高级感，参考 Midjourney / Runway
- **主色调**：#6c5ce7（紫），强调色：#00cec9（青）
- **交互**：全站无刷新，WebSocket 实时推送状态
