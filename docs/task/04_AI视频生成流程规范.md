# AI 视频生成流程规范（pi-mono Agent Pipeline）

> 本文档定义 7 个 Agent 的详细输入输出规范、数据格式、以及 Agent 间数据传递规则。
> pi-mono 长期记忆的使用规范也包含在内。

---

## 通用规则

### 数据传递
- 每个 Agent 输出 JSON 格式的 `dict`，作为下一个 Agent 的 `input_data` 的一部分
- 每个 Agent 输出的 key 不允许与输入 key 冲突（除非明确覆盖）
- 所有 Agent 输出必须包含字段：`{ "agent_name": "...", "timestamp": "...", "data": {...} }`

### pi-mono Memory 使用
- `session_id`：一次完整的 story 生成会话，唯一标识
- `story_id`：一个 story 的唯一标识
- 每个 Agent **执行前** 检索记忆：`memory.search(session_id, query, top_k=3)`
- 每个 Agent **执行后** 写入记忆：`memory.add(session_id, story_id, agent_name, output)`
- 检索 query 策略：
  - 使用当前 Agent 输入中的 `story_summary` 或 `description`
  - 跨 story 检索：`session_id` 相同时可跨 story 检索同类型情节/角色的记忆

### 人工干预注入
- `human_feedback` 非空时，Agent prompt 末尾追加：`"\n\n【人工修改建议】：{human_feedback}\n请根据此建议重新生成输出。"`
- Agent 重新执行时应保留原输出的有效部分，只根据 feedback 调整有问题的部分

## Agent 详细规范

### Agent 1：ScriptAnalysis（剧本分析）

**文件**：`app/agents/script_analysis.py`

**职责**：理解 story 文本，提取核心信息，识别类型和风格

**输入**：
```json
{ "story_text": "string（原始 story 全文）" }
```

**输出**：
```json
{
  "agent_name": "ScriptAnalysis",
  "timestamp": "ISO8601",
  "data": {
    "story_summary": "string（200字以内的故事概要）",
    "genre": "string（剧情/科幻/奇幻/爱情/悬疑/惊悚/动画等）",
    "sub_genre": "string（可选，子类型）",
    "tone": "string（整体基调：轻松/紧张/悬疑/感人/热血等）",
    "target_audience": "string（目标受众年龄和群体）",
    "themes": ["string（核心主题列表）"],
    "mood_tags": ["string（氛围标签，用于视觉生成参考）"],
    "estimated_duration_sec": "number（根据故事长度估算的视频时长）",
    "language": "string（语言，默认 zh-CN）"
  }
}
```

**Mock → Real 升级路径**：
- Mock：正则 + 规则匹配输出
- Real：调用 LLM（如 GPT-4o / Claude）做摘要和分类

### Agent 2：PlotDeconstruct（情节拆解）

**文件**：`app/agents/plot_deconstruct.py`

**职责**：将故事拆解为三幕结构，识别关键情节点

**输入**：
```json
{
  "story_text": "string",
  "story_summary": "string（来自 Agent1）",
  "genre": "string",
  "tone": "string"
}
```

**输出**：
```json
{
  "agent_name": "PlotDeconstruct",
  "timestamp": "ISO8601",
  "data": {
    "act_structure": {
      "act1": {
        "title": "string（标题）",
        "description": "string（描述，50-100字）",
        "ending_hook": "string（悬念/转折点）"
      },
      "act2": {
        "title": "string",
        "description": "string",
        "rising_action": ["string（递进情节点）"],
        "midpoint Twist": "string（中点转折）"
      },
      "act3": {
        "title": "string",
        "description": "string",
        "climax": "string（高潮）",
        "resolution": "string（结局）"
      }
    },
    "key_plot_points": [
      {
        "point_id": "number",
        "type": "string（setup/conflict/climax/resolution）",
        "description": "string",
        "emotion": "string（该情节点引发的情感）"
      }
    ],
    "conflict": "string（核心冲突描述）",
    "pacing": "string（节奏评估：快/中等/慢）"
  }
}
```

### Agent 3：ScenePlanning（分镜规划）

**文件**：`app/agents/scene_planning.py`

**职责**：基于情节结构，规划具体分镜序列

**输入**：
```json
{
  "act_structure": { ...（来自 Agent2） },
  "key_plot_points": [...],
  "genre": "string",
  "estimated_duration_sec": "number"
}
```

**输出**：
```json
{
  "agent_name": "ScenePlanning",
  "timestamp": "ISO8601",
  "data": {
    "scenes": [
      {
        "scene_id": "string（sc_01, sc_02...）",
        "scene_number": "number（序号）",
        "act": "number（属于第几幕）",
        "description": "string（分镜描述，30-80字）",
        "location": "string（场景地点：室内/室外 + 具体场景）",
        "time_of_day": "string（day/night/dusk/dawn/morning）",
        "weather": "string（可选：sunny/rainy/cloudy/snowy）",
        "emotion": "string（该场景传达的情感）",
        "camera_angle": "string（镜头建议：wide/medium/close-up/drone等）",
        "transition": "string（切换方式：cut/fade/dissolve/s wipe）",
        "duration_sec": "number（该场景预估时长）",
        "depends_on_scenes": ["string（前置场景 ID，可选）"]
      }
    ],
    "total_scenes": "number",
    "estimated_total_duration": "number"
  }
}
```

### Agent 5：VisualGen（视觉生成）

**文件**：`app/agents/visual_gen.py`

**职责**：为每个分镜生成视觉资产（图像或视频片段），以及角色定妆照

**输入**：
```json
{
  "scenes": [...（来自 Agent3）],
  "characters": [...（来自 Agent4）],
  "genre": "string",
  "mood_tags": [...]
}
```

**输出**：
```json
{
  "agent_name": "VisualGen",
  "timestamp": "ISO8601",
  "data": {
    "visual_assets": [
      {
        "asset_id": "string（asset_01...）",
        "scene_id": "string（对应的场景 ID）",
        "character_id": "string（可选，对应的角色 ID）",
        "asset_type": "string（image/video_clip/background）",
        "prompt": "string（AI 图像/视频生成 prompt，英文）",
        "negative_prompt": "string（反向 prompt）",
        "generation_params": {
          "model": "string（使用的模型，如 stable-diffusion-xl）",
          "resolution": "string（如 1920x1080）",
          "seed": "number（随机种子）",
          "steps": "number（推理步数）"
        },
        "output_path": "string（本地或 OSS 路径）",
        "output_url": "string（访问 URL）",
        "status": "string（pending/generating/done/failed）",
        "retry_count": "number（重试次数）"
      }
    ],
    "character_sheets": [
      {
        "character_id": "string",
        "asset_id": "string",
        "output_path": "string"
      }
    ]
  }
}
```

**生成策略**：
- 同一 scene 的多个镜头可并行生成（利用 `asyncio.gather`）
- 调用 `VideoGeneratorService.generate_image(prompt)` 或 `generate_video_clip(prompt)`
- 生成失败自动重试 2 次，超时报错

---

### Agent 6：VideoAssembly（视频剪辑合成）

**文件**：`app/agents/video_assembly.py`

**职责**：将所有视觉资产按分镜顺序组装为完整视频

**输入**：
```json
{
  "visual_assets": [...],
  "scenes": [...],
  "characters": [...（用于字幕/人名贴片）],
  "mood_tags": [...]
}
```

**输出**：
```json
{
  "agent_name": "VideoAssembly",
  "timestamp": "ISO8601",
  "data": {
    "final_video_path": "string",
    "final_video_url": "string",
    "duration_sec": "number",
    "resolution": "string（如 1920x1080）",
    "codec": "string（如 h.264）",
    "fps": "number（如 24）",
    "assembly_edl": "string（剪辑决策列表，EDL 格式文本，用于可解释性）",
    "scenes_in_order": ["string（最终视频中场景的顺序）"],
    "file_size_mb": "number"
  }
}
```

**技术方案**：
- 调用 `VideoAssemblyService.assemble(assets, scenes)` 
- 服务内部调用 FFmpeg 或对接视频剪辑 API
- 若视频生成 API 支持直接输出视频片段（video_clip），则按 scene 顺序拼接

### Agent 7：QCReview（质检复审）

**文件**：`app/agents/qc_review.py`

**职责**：检查最终视频质量，输出问题报告（不阻塞主流程，异步执行）

**输入**：
```json
{
  "final_video_path": "string",
  "final_video_url": "string",
  "scenes": [...],
  "characters": [...],
  "story_summary": "string"
}
```

**输出**：
```json
{
  "agent_name": "QCReview",
  "timestamp": "ISO8601",
  "data": {
    "quality_score": "number（0-100）",
    "issues": [
      {
        "severity": "string（critical/major/minor）",
        "type": "string（audio/video/consistency/pace）",
        "description": "string",
        "affected_scene_id": "string（可选）",
        "suggestion": "string（修复建议）"
      }
    ],
    "recommendations": ["string（整体优化建议）"],
    "passed": "boolean（是否通过质检）"
  }
}
```

**实现说明**：
- 异步执行，不阻塞 `VideoAssembly` 之后的流程
- 可调用视频分析 LLM API（如 GPT-4o vision）分析关键帧
- 结果通过 WebSocket `qc_complete` 消息推送前端

## 完整数据流示例

```
story_text
    ↓
[Agent1: ScriptAnalysis] → story_summary, genre, tone, themes...
    ↓
[Agent2: PlotDeconstruct] → act_structure, key_plot_points, conflict
    ↓
[Agent3: ScenePlanning]  → scenes[], total_duration
    ↓              ↕（并行）
[Agent4: CharacterDesign] → characters[], narrator
    ↓
[Agent5: VisualGen]      → visual_assets[], character_sheets
    ↓
[Agent6: VideoAssembly]  → final_video_path, duration
    ↓（异步）
[Agent7: QCReview]       → quality_score, issues[]
```

## 错误处理规范

| 错误场景 | 处理策略 |
|---------|---------|
| 单个 Agent 执行失败 | 重试 2 次 → 失败则 story status = error，WebSocket 推送 error 消息 |
| 人工干预超时（30min） | 自动确认，继续下一 Agent |
| 视频生成 API 不可用 | 降级到本地 FFmpeg 方案（需预先准备 fallback 素材） |
| 向量数据库连接失败 | 降级到纯内存模式（无长期记忆，但 pipeline 继续运行） |
