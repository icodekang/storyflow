# StoryFlow AI 视频生成项目 — 开发任务总览

> 项目目标：输入 story 文本，自动调度多个 AI Agent 协作，最终输出 AI 生成的视频。
> 核心参考：电影制作流程（前期策划 → 分镜制作 → 角色设计 → 视觉生成 → 后期剪辑 → 合成输出）

---

## 二、前端开发任务

### 2.1 页面结构（无登录）

```
/               → 主工作台（无需认证）
  ├── Story 输入区（支持多 story 卡片）
  ├── 模式切换（全自动 / 人工干预）
  ├── Agent 流程可视化面板
  ├── 当前 Agent 干预抽屉（人工干预模式）
  └── 视频预览 / 下载区
```

### 2.2 核心功能

#### 2.2.1 Story 输入区
- **多 story 并发输入**：每个 story 为一张卡片，支持"新增 story"按钮动态添加
- 每个卡片字段：
  - `story_id`（系统自动生成，UUID）
  - `story_text`（多行文本输入，支持粘贴长文本）
  - `genre`（下拉：剧情 / 科幻 / 奇幻 / 爱情 / 悬疑，默认识别）
  - `style`（下拉：写实 / 动漫 / 3D CGI，默认识别）
  - `duration`（目标时长，秒，默认识别）
  - `status`（待生成 / 生成中 / 完成 / 失败）
- **开始生成按钮**：批量启动选中 story 的生成流程

#### 2.2.2 模式切换
- 页面顶部全局开关：**全自动模式** / **人工干预模式**
  - **全自动**：所有 Agent 串联执行，无需人工介入
  - **人工干预**：每个 Agent 完成后暂停，等待人工确认
- 干预选项：
  - **确认流转**：直接进入下一个 Agent
  - **修改建议**：输入修改意见，重新让当前 Agent 生成
  - **跳过此步**：跳过当前 Agent，使用默认结果

#### 2.2.3 Agent 流程可视化
- 横向流程图展示 7 个 Agent 阶段（见后端 pipeline）
- 每个节点显示：
  - Agent 名称 + 图标
  - 状态（pending / running / waiting_intervention / done / error）
  - 耗时
- 当前运行节点高亮，完成节点打勾，错误节点标红

#### 2.2.4 人工干预抽屉（人工干预模式）
- 当 Agent 等待干预时，从右侧滑出抽屉
- 显示内容：
  - 当前 Agent 名称 + 任务描述
  - Agent 生成的原始输出（结构化展示，如分镜文本、角色描述等）
  - 修改建议文本框（可选）
  - 三个按钮：【确认流转】【重新生成】【跳过】
- 重新生成时显示 loading spinner，完成后更新预览

#### 2.2.5 视频预览与下载
- 生成完成后，显示最终视频播放器
- 支持预览和下载（`.mp4`）
- 多 story 时，每个 story 独立视频，独立卡片展示

#### 2.2.6 UI 设计规范
- **整体风格**：深色主题（类似 Midjourney / Runway），极简高级
- **配色**：
  - 背景：`#0a0a0f`（近黑）
  - 卡片：`#14141f`（深灰紫）
  - 主色调：`#6c5ce7`（紫）
  - 强调色：`#00cec9`（青）
  - 文字主色：`#ffffff`，次要：`#8892b0`
- **字体**：Inter / Noto Sans SC（中文）
- **交互**：
  - 所有操作无需页面刷新
  - 状态更新通过 WebSocket 实时推送（优先）或轮询
  - 卡片添加/删除带平滑动画
  - 流程节点点击可展开详情

## 三、后端开发任务

### 3.1 技术选型
- **框架**：FastAPI + Python 3.11+
- **Agent 框架**：参考 LangChain / LangGraph 设计 Agent，核心基于 **pi-mono**
- **长期记忆**：pi-mono Memory Module（向量数据库：Milvus / Qdrant / ChromaDB，可配置）
- **异步任务**：Celery + Redis（任务队列），或 FastAPI BackgroundTasks
- **WebSocket**：用于向前端推送 Agent 执行状态
- **视频生成**：对接外部视频生成 API（如 Runway、Pika、本地模型），抽象为 `VideoGenerator` 接口
- **存储**：本地文件系统（开发）或 OSS（生产），抽象为 `StorageService`

### 3.2 Agent Pipeline（电影制作流程）

每个 Agent 接收上一阶段的输出，叠加记忆上下文，产出自己的结果：

```
① ScriptAnalysis Agent
   输入：story_text
   输出：{ story_summary, genre, tone, target_audience, tags[] }

② PlotDeconstruct Agent
   输入：story_summary, story_text
   输出：{ act_structure{act1, act2, act3}, key_plot_points[], conflict, resolution }

③ ScenePlanning Agent
   输入：act_structure, key_plot_points
   输出：{ scenes[]{ scene_id, description, location, time_of_day, emotion, estimated_duration } }

④ CharacterDesign Agent
   输入：story_text, key_plot_points
   输出：{ characters[]{ name, role, appearance, personality, voice_description, relationships[] } }

⑤ VisualGen Agent
   输入：scenes[], characters[]
   输出：{ visual_assets[]{ asset_id, scene_id, asset_type(image/video_clip), prompt, generation_params, output_path } }

⑥ VideoAssembly Agent
   输入：visual_assets[], scenes[], characters[]
   输出：{ final_video_path, assembly_edl, duration, resolution, codec }

⑦ QCReview Agent（可选 / 自动）
   输入：final_video_path
   输出：{ quality_score, issues[], recommendations[] }
   注意：此 Agent 不阻塞主流程，结果异步回写
```

### 3.3 Agent 编排引擎

#### Pipeline Orchestrator
- 管理完整的 Agent 链式调用
- 支持：**串行执行**（默认）、**并行分支**（如 VisualGen 可并行生成多个场景）
- 支持**条件路由**：根据 story 类型跳过或替换某些 Agent
- 每次 Agent 调用携带 `session_id` + `story_id` 查询长期记忆

#### Human-in-the-loop Controller
- 人工干预模式下，Agent 完成后写入状态 `waiting_intervention` 到 Redis/DB
- 通过 WebSocket 通知前端
- 等待前端回调（确认 / 修改意见 / 跳过）
- 收到修改意见时，将意见作为 `human_feedback` 注入 Agent 重新执行

### 3.4 pi-mono Long-term Memory 集成

- **Memory Service** 提供以下接口：
  - `memory.add(session_id, story_id, agent_name, content)` — 写入记忆
  - `memory.search(session_id, query, top_k)` — 相似记忆检索
  - `memory.get_history(session_id, story_id)` — 获取某 story 的完整记忆链
- 每次 Agent 执行前，先 `memory.search` 获取相关上下文，拼入 prompt
- 记忆按 session 分隔，支持跨 story 检索同类型情节、角色的记忆

### 3.5 API 接口设计

```
POST /api/stories              → 创建 story 任务（body: { stories: [{text, genre, style}] }）
GET  /api/stories/:id          → 获取 story 任务状态
GET  /api/stories/:id/agents  → 获取各 Agent 输出结果
POST /api/stories/:id/generate → 触发生成流程（body: { mode: 'auto' | 'human' }）

GET  /api/agents/:id/output    → 获取单个 Agent 输出
POST /api/agents/:id/intervene → 人工干预（body: { action: 'confirm' | 'regenerate' | 'skip', feedback?: string }）

WebSocket /ws/stories/:id      → 实时推送 pipeline 状态更新
```

### 3.6 多 Story 并发支持

- 每个 story 独立 `session_id`，独立 pipeline 实例
- 后端维护全局任务队列（Celery），按配置并发数调度
- 前端展示所有 story 卡片状态，独立刷新
- 支持在生成过程中动态新增 story（入队，不影响正在运行的任务）

## 五、开发任务优先级

### P0（核心 MVP）
1. 后端项目初始化（FastAPI + 目录结构）
2. Agent 基类 + Pipeline Orchestrator 实现
3. 7 个 Agent 的骨架实现（可先返回 mock 数据）
4. 前端主页面框架 + Story 输入卡片
5. 全自动模式端到端串联（前后端联调）
6. WebSocket 状态推送

### P1（完整功能）
7. 人工干预模式完整实现
8. pi-mono Memory 集成（向量数据库选型）
9. 多 story 批处理
10. 视频预览 + 下载

### P2（体验优化）
11. QCReview Agent 实现
12. 干预抽屉详情展示优化
13. UI 细节打磨（动画、状态管理）
14. 视频生成服务抽象（支持多后端切换）
