> 对应后端 API 文档：见 `02_后端开发任务.md`

## F1：项目初始化

**目标**：搭建前端工程，配置基础样式系统和组件库

**步骤**：
1. 使用 Vite + React + TypeScript 初始化项目
   ```bash
   cd storyflow/frontend
   npm create vite@latest . -- --template react-ts
   ```
2. 安装依赖：
   - `antd`（UI 组件库）或 `shadcn/ui`（更轻量自定义强）
   - `zustand`（状态管理）
   - `@tanstack/react-query`（数据请求）
   - `react-router-dom`（路由）
   - `zustand`（全局状态）
   - `lucide-react`（图标库）
3. 配置 Tailwind CSS（深色主题配置）
4. 在 `src/styles/globals.css` 中定义 CSS 变量（参考总览配色方案）
5. 配置 `vite.config.ts` 支持 WebSocket 代理到后端

**验收**：
- `npm run dev` 启动无报错
- 页面显示基础深色背景

## F2：Story 输入卡片组件

**文件**：`src/components/StoryInput/`

**需求**：
- 每个 Story 是一张可折叠/展开的卡片
- 卡片内字段：story_text（textarea）、genre（select）、style（select）、duration（number，可选）
- 每个卡片顶部显示 status badge（待生成 / 生成中 / 完成 / 失败）+ 耗时
- 卡片右上角删除按钮（删除此 story）
- 底部"新增 Story"按钮，点击添加新卡片
- 支持至少 3 个 story 同时展示

**接口对接**：
- 卡片 mount 时调用 `GET /api/stories` 获取已有 story 列表
- 点击"开始生成"调用 `POST /api/stories/:id/generate`

**验收**：
- 能添加多个 story 卡片
- 能删除单个卡片
- status 实时更新（WebSocket 推送或轮询）

## F4：Agent 流程可视化

**文件**：`src/components/PipelineView/`

**需求**：
- 横向 7 步流程图：① 剧本分析 → ② 情节拆解 → ③ 分镜规划 → ④ 角色设计 → ⑤ 视觉生成 → ⑥ 视频剪辑 → ⑦ 质检
- 每个节点状态样式：
  - pending：灰色圆角方块
  - running：紫色脉冲动画
  - waiting_intervention：橙色边框（仅人工干预模式）
  - done：绿色打勾
  - error：红色感叹号
- 点击节点展开详情（浮层或抽屉），显示该 Agent 的输入/输出文本
- 流程图下方显示当前步骤耗时

**验收**：
- 能正确显示 7 个节点
- 状态变化时实时高亮当前节点

## F5：人工干预抽屉

**文件**：`src/components/InterventionDrawer/`

**需求**（仅人工干预模式）：
- 当 Agent 处于 `waiting_intervention` 状态时，从右侧滑出
- 抽屉内容：
  - 当前 Agent 名称（大字）+ 任务描述（次要文字）
  - Agent 输出内容预览（markdown 或结构化 JSON 渲染）
  - "修改建议" 文本框（placeholder：你希望如何修改？）
  - 三个操作按钮：【确认流转】【重新生成】【跳过此步】
- 点击【重新生成】：调用干预接口，loading 显示 Agent 重新执行
- 点击【确认流转】：调用确认接口，关闭抽屉，pipeline 继续
- 点击【跳过此步】：调用跳过接口，关闭抽屉，pipeline 继续

**WebSocket 监听**：
- 监听 `/ws/stories/:id`，收到 `agent_waiting` 事件时自动弹出抽屉

**验收**：
- 正确弹出、关闭
- 三个操作都正确调用接口
- 重新生成时显示 loading，完成后更新 Agent 输出

## F6：视频预览与下载

**文件**：`src/components/VideoPreview/`

**需求**：
- 生成完成后，在 Story 卡片下方展开视频播放器
- 使用 `<video>` 标签，支持播放、暂停、音量
- 视频下方显示"下载视频"按钮，下载为 `.mp4`
- 若生成失败，显示错误原因 + "重新生成"按钮

**验收**：
- 视频可正常播放
- 下载按钮触发浏览器下载

## F7：多 Story 批处理管理

**文件**：`src/components/StoryList/` + `src/stores/storyStore.ts`

**需求**：
- 全局状态管理（Zustand store）维护所有 story 的状态列表
- 每个 story 的 `status`、`current_agent`、`progress` 独立更新
- 支持"全部开始"一键触发所有待生成的 story
- 生成中的 story 显示独立进度

**验收**：
- 同时运行 3 个 story，前端无卡顿
- 各自状态独立，互不干扰

## F8：WebSocket 实时状态

**文件**：`src/services/websocket.ts`

**需求**：
- 连接 `ws://localhost:8000/ws/stories/:id`
- 解析消息类型：
  - `agent_status` → 更新 PipelineView 中对应节点状态
  - `agent_output` → 更新 Agent 输出内容
  - `agent_waiting` → 弹出干预抽屉
  - `story_complete` → 更新 story 卡片状态 + 显示视频
  - `story_error` → 更新 story 卡片状态 + 显示错误
- 自动重连（指数退避）
- 连接状态指示（底部小圆点：绿色=已连接，灰色=断开）

**验收**：
- Agent 每完成一步，前端流程图节点即时更新
- 无需手动刷新页面

## F9：UI 细节打磨

**非阻塞，建议最后统一处理**：
- 深色主题一致性（全局 CSS 变量覆盖 antd 默认色）
- 卡片添加/删除动画（Framer Motion 或 CSS transition）
- PipelineView 节点 hover 微动效
- 加载状态：Skeleton 骨架屏（不要用 spinner 占满屏）
- 空状态：没有任何 story 时显示引导插图 + "添加你的第一个 Story"按钮
- 响应式：支持 1280px 以上宽屏（主要场景）
