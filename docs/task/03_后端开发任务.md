## B2：Agent 基类

**文件**：`app/agents/base.py`

**目标**：定义所有 Agent 的抽象基类，统一接口

```python
class BaseAgent(ABC):
    name: str
    description: str

    async def execute(
        self,
        session_id: str,
        story_id: str,
        input_data: dict,
        memory: MemoryService,
        human_feedback: str | None = None,
    ) -> dict:
        """执行 Agent 逻辑，返回输出数据字典"""
        raise NotImplementedError
```

**要求**：
- 所有 Agent 继承 `BaseAgent`
- `execute` 为异步方法，内部先调用 `memory.search` 获取相关记忆
- 执行完成后调用 `memory.add` 写入本次输出到记忆
- 如果 human_feedback 存在，将其加入 prompt 重新执行

**验收**：
- 7 个 Agent 都继承此基类
- 抽象方法定义清晰，子类只需实现 `execute` 逻辑

## B3：Pipeline Orchestrator

**文件**：`app/core/orchestrator.py`

**目标**：管理 Agent 链式执行，支持全自动和人工干预两种模式

**核心逻辑**：
```python
class PipelineOrchestrator:
    def __init__(self, mode: Literal["auto", "human"]):
        self.mode = mode
        self.agents = [ScriptAnalysisAgent(), PlotDeconstructAgent(), ...]

    async def run(self, story_id: str, story_data: dict, websocket: WebSocket):
        """执行完整 pipeline，通过 websocket 推送状态"""
        for agent in self.agents:
            await websocket.send_json({"type": "agent_status", "agent": agent.name, "status": "running"})
            output = await agent.execute(session_id=..., story_id=..., ...)

            if self.mode == "human":
                # 写入等待状态，发消息给前端
                await websocket.send_json({"type": "agent_waiting", "agent": agent.name, "output": output})
                # 等待前端干预回调（见 B5）
                intervention = await wait_for_intervention(story_id, agent.name)
                if intervention["action"] == "regenerate":
                    output = await agent.execute(..., human_feedback=intervention["feedback"])
                elif intervention["action"] == "skip":
                    output = DEFAULT_OUTPUT[agent.name]
            else:
                await websocket.send_json({"type": "agent_status", "agent": agent.name, "status": "done"})

            await websocket.send_json({"type": "agent_output", "agent": agent.name, "output": output})
```

**要求**：
- 人工干预模式下，使用 Redis（或其他队列）存储 `story_id:agent_name → intervention_result` 键值对
- 前端回调接口写入该键，orchestrator 轮询或通过事件等待该键被设置
- 每个 Agent 执行后推送 WebSocket 状态

**验收**：
- 全自动模式：7 个 Agent 顺序执行，无阻塞
- 人工干预模式：Agent 完成后正确暂停

## B5：人工干预 API

**文件**：`app/api/intervention.py`

**接口**：
```
POST /api/agents/{story_id}/{agent_name}/intervene
Body: { "action": "confirm" | "regenerate" | "skip", "feedback"?: string }
```

**逻辑**：
1. 将干预结果写入 Redis：`intervention:{story_id}:{agent_name}` = json(body)
2. 返回 200

**Orchestrator 端**：
- 在 `wait_for_intervention()` 函数中轮询 Redis 键或订阅 Pub/Sub
- 超时时间（如 30 分钟），超时则自动 confirm

## B6：API 路由实现

### Stories API
```
POST /api/stories
  Body: { stories: [{text, genre, style, duration}] }
  返回: { stories: [{id, status}] }

GET /api/stories/{id}
  返回: Story 完整信息 + 所有 Agent 输出

GET /api/stories/{id}/agents
  返回: 各 Agent 名称、状态、输出摘要
```

### Generate API
```
POST /api/stories/{id}/generate
  Body: { mode: "auto" | "human" }
  启动 orchestrator.run()（后台 task）
  返回: { message: "started", task_id }
```

### Agent Output API
```
GET /api/agents/{story_id}/{agent_name}/output
  返回: 该 Agent 的完整输出
```

class VideoGenerator(ABC):

## B10：多 Story 并发调度

**目标**：通过 Celery 或 asyncio 实现多 story 并发生成

**方案（推荐 asyncio 方案，轻量）**：
```python
# app/core/task_queue.py
import asyncio

class TaskQueue:
    def __init__(self, max_concurrent: int = 3):
        self.semaphore = asyncio.Semaphore(max_concurrent)
        self.tasks: dict[str, asyncio.Task] = {}

    async def enqueue(self, story_id: str, coro):
        async with self.semaphore:
            self.tasks[story_id] = asyncio.create_task(coro)
            await self.tasks[story_id]

    def cancel(self, story_id: str):
        if story_id in self.tasks:
            self.tasks[story_id].cancel()
```

**要求**：
- 并发数可配置（`app/core/config.py` → `MAX_CONCURRENT_STORIES`）
- 新 story 加入时检查是否超过并发上限，超则排队
